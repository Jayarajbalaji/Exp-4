# Exp - 4 HOSPITAL MANAGEMENT SYSTEM
# AIM

To study and understand the functionalities, workflow, and requirements of a Hospital Management System using Software Requirement Specification (SRS).

# SRS (Procedure)
# 1. Introduction

A Hospital Management System (HMS) is software designed to manage hospital operations such as patient registration, doctor appointments, billing, medical records, pharmacy, and ward management.

# 2. Purpose

The purpose of this system is to:

Maintain patient and doctor records

Automate appointment and treatment processes

Manage pharmacy and billing

Improve accuracy, efficiency, and workflow inside the hospital

# 3. Scope

The system covers:

Patient registration and admission

Doctor appointments and schedules

Inpatient and outpatient management

Prescription and pharmacy management

Billing and payment

Staff and administrative operations

Reports and medical history

# 4. Functional Requirements

System must store patient personal and medical details.

Doctors can update patient diagnosis and treatment.

Receptionist can schedule and cancel appointments.

Pharmacy manages medicine stock and prescriptions.

Billing module generates patient bills.

Admin manages users, staff, and system configuration.

# 5. Non-Functional Requirements

Security: Patient data confidentiality must be maintained.

Usability: System must be simple for hospital staff.

Performance: Fast access to patient records and doctor schedules.

Reliability: Accurate medical and billing data.

Availability: System should support 24/7 operation.

# 6. Assumptions

Each patient is given a unique hospital ID.

Hospital staff have different access levels.

All modules (pharmacy, billing, reception) are interconnected.
